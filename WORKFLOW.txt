───────────────────────────────────────────────────────────────
SETUP (one time only — already done)
───────────────────────────────────────────────────────────────

Repository : https://github.com/guillaume-bruelle/raiz-dst
Live URL   : https://guillaume-bruelle.github.io/raiz-dst/
Local path : wherever you cloned raiz-dst/ on your computer

Tools used:
  - VS Code           → editing, committing and pushing to GitHub
  - Claude Code       → AI assistance for code changes (VS Code panel)
  - Chrome/Firefox    → testing the tool locally


───────────────────────────────────────────────────────────────
EVERY TIME YOU SIT DOWN TO WORK
───────────────────────────────────────────────────────────────

1. Open VS Code
   → File → Open Folder → select your raiz-dst/ folder

2. Pull latest changes: open Source Control (Ctrl+Shift+G)
   → Click the "..." menu → Pull

3. Open the Claude Code panel in VS Code (sidebar)
   → Have app/raiz_dst.html open alongside

You are ready to work.


───────────────────────────────────────────────────────────────
MAKING A CHANGE (small fix or improvement)
───────────────────────────────────────────────────────────────

1. In VS Code, open app/raiz_dst.html

2. Use Ctrl+F to find the right block by its label, e.g.:
     js-engine        → calculation formulas
     js-entry         → data entry form logic
     js-results       → results rendering and charts
     js-sessions      → save/load/export logic
     js-projects      → project management
     js-calendar      → period definitions
     js-crop-products → crop product parameters
     js-member-types  → household member types
     js-cropping-systems → cropping system definitions
     js-farms         → farm profile management
     js-scenarios     → scenario simulation
     js-nav           → navigation between tabs
     js-state         → shared data and autosave
     js-utils         → helper functions (_esc, _download)

3. Open the Claude Code panel in VS Code (sidebar)
   → Describe what you want changed, e.g.:
       "In the js-engine block, update the food purchase
        distribution to be weighted by period length
        instead of split equally across periods"
   → Claude will read the file, make the edit directly, and
     show you a diff to review before applying

4. Review the diff in the editor
   → Accept if it looks correct

5. Save the file (Ctrl+S)

6. Test it: open app/raiz_dst.html in Chrome
   → Drag the file into Chrome, or right-click → Open with Chrome

7. If it works → commit (see COMMITTING below)
   If not → describe the problem to Claude in the same panel


───────────────────────────────────────────────────────────────
COMMITTING AND PUSHING
───────────────────────────────────────────────────────────────

After any change that works:

1. Open Source Control in VS Code (Ctrl+Shift+G)
   → You will see raiz_dst.html listed as changed

2. Click the "+" next to the file to stage it

3. Type a commit message in the box at the top.
   Use this format:

     fix: land bar not updating when farm area changes
     feat: add livestock section to data entry
     model: update labour supply proration formula
     docs: add consumption note to user guide
     style: improve mobile layout for data entry tab
     release: v1.1

   Prefix guide:
     fix:     bug fix, no new feature
     feat:    new feature or UI addition
     model:   change to calculation engine
     docs:    change to guide or documentation files
     style:   visual or layout change only
     release: new version snapshot

4. Click "Commit" (checkmark button)

5. Click "Sync Changes" (or "Push") to push to GitHub

6. Done. The live URL updates automatically within ~2 minutes.
   Check: https://guillaume-bruelle.github.io/raiz-dst/


───────────────────────────────────────────────────────────────
RELEASING A NEW VERSION
───────────────────────────────────────────────────────────────

When you have made enough changes to warrant a new version:

Version numbering:
  v1.0.1  → small bug fix, model unchanged
  v1.1    → new feature, model unchanged
  v2.0    → model equations or data structure changed

Steps:

1. In VS Code, open app/raiz_dst.html
   → Find the <title> tag at the top
   → Update: <title>RAIZ — Farm Analysis DST v1.1</title>

2. Copy app/raiz_dst.html
   → Paste into versions/
   → Rename to: v1.1_raiz_dst.html

3. Open CHANGELOG.md
   → Add a new entry at the top:

     ## [v1.1] — 2025-12-01
     ### Added
     - Brief description of what changed

4. Commit with message: release: v1.1

5. Push


───────────────────────────────────────────────────────────────
SHARING THE TOOL WITH FIELD AGENTS
───────────────────────────────────────────────────────────────

Option A — Live URL (easiest, requires internet once)
  Share this link:
  https://guillaume-bruelle.github.io/raiz-dst/
  → Works on any browser, desktop or Android
  → Always the latest version

Option B — Offline file (no internet needed after transfer)
  1. Go to github.com/guillaume-bruelle/raiz-dst
  2. Open app/raiz_dst.html
  3. Click "Download raw file"
  4. Send the file by email, USB, or WhatsApp
  → Agent opens it in Chrome on their device
  → Works fully offline

Sharing data between agents (sessions):
  Agent A: Sessions & Export → Download .json
  Send the .json file to Agent B (email, WhatsApp, USB)
  Agent B: Sessions & Export → Load from file → select .json


───────────────────────────────────────────────────────────────
SAVING AND BACKING UP FIELD DATA
───────────────────────────────────────────────────────────────

The tool auto-saves to browser localStorage (see timestamp
in the top-right header after every change).

But localStorage is NOT a permanent backup. Always:

  End of each field day:
  → Sessions & Export tab
  → Give session a name: e.g. Mutoko_2025-11-21
  → Click "Save to Browser" (checkpoint in this browser)
  → Click "Download .json" (real file backup on device)
  → Store the .json in a safe folder (cloud drive, USB)

  For the research team:
  → Sessions & Export tab → Export Farm Data for Research
  → Check status panel: all farms should show ✓
  → Click "Download CSV"
  → Send CSV to research team for analysis
